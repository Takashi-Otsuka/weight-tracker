# Comment Policy

## Purpose

本ドキュメントは、本プロジェクトにおけるコードコメント方針を定義する。

目的:

- コードの可読性を保つ
- 不要なコメントを防ぐ
- Codexが意図を理解しやすくする
- 将来の保守性を高める

------------------------------------------------------------------------

# Basic Principle

コードを読めば分かる内容はコメントしない。

コメントは「何をしているか」ではなく、「なぜそうしているか」を説明するために書く。

------------------------------------------------------------------------

# Write Comments For

## Why

実装理由や設計判断を書く。

``` ts
// localStorage破損時は空配列へフォールバックする。
// UIクラッシュ防止を優先する。
return [];
```

## Business Rule

仕様・業務ルールを書く。

``` ts
// 体重記録は1日1件とする。
// 同一日付の記録は上書きする。
```

## Constraint

技術的制約や回避策を書く。

``` ts
// iOS Safariでの日付解釈差異を避けるため、YYYY-MM-DD文字列として扱う。
```

## TODO

未対応事項を書く。

``` ts
// TODO: Supabase移行時にRepository層へ移動する。
```

------------------------------------------------------------------------

# JSDoc

外部から利用される関数にはJSDocを推奨する。

対象:

- `src/lib/**`
- `src/features/**`

例:

``` ts
/**
 * YYYY-MM-DD形式の今日の日付を返す。
 */
export function getToday(): string {
  // ...
}
```

------------------------------------------------------------------------

# Prohibited Comments

## Code Description Only

``` ts
// 体重を保存する
saveWeightRecord(record);
```

## Commented-out Code

``` ts
// const oldValue = getOldValue();
```

## Author Name

``` ts
// Takashiが追加
```

## Date-based History

``` ts
// 2026/05/30 修正
```

変更履歴はGitで管理する。

------------------------------------------------------------------------

# TODO Rule

許可するタグ:

- `TODO:`
- `FIXME:`
- `HACK:`

書式:

``` ts
// TODO: 理由を書く
// FIXME: 修正が必要な理由を書く
// HACK: 一時対応である理由を書く
```

理由のないTODOは禁止する。

------------------------------------------------------------------------

# Codex Rule

Codexは以下を守ること。

- 不要な説明コメントを追加しない
- コードで表現できる内容をコメントしない
- 業務ルール・制約・TODOはコメントとして残す
- JSDocは公開関数に限定する
- コメントアウトコードを残さない
